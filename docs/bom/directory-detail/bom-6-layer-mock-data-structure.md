# BOM 6階層モックデータ構造案

## 概要
- **目的**: Directory詳細ページのBOMツリー表示用モックデータ
- **階層数**: 6階層
- **特徴**: Directory, LeafProduct, Documentが混在する複雑な構造

## データベーススキーマとの対応

### Directoryに紐づくDocument (directory_documents)
- 📄 アイコンで表示
- Directory直下に配置される文書・仕様書類
- 例: 製品仕様書、安全基準書、取扱説明書、組立手順書など

### LeafProductに紐づくファイル
#### 図面 (drawing_files)
- 📐 アイコンで表示
- 2D図面ファイル (DWG, PDF等)
- **全てのLeafProductに以下を含める:**
  - 📐 図面1
  - 📐 図面2
  - 📐 図面3

#### ドキュメント (leaf_product_documents)
- 📄 アイコンで表示
- 部品に関する文書類
- **全てのLeafProductに以下を含める:**
  - 📄 仕様書
  - 📄 3Dモデル

## 統計情報

### 階層別統計
- **第1階層（製品）**: 1個
- **第2階層（Assy）**: 5個
- **第3階層（SubAssy）**: 17個
- **第4階層（SubSubAssy）**: 18個
- **第5階層（Module）**: 8個
- **第6階層（LeafProduct）**: 約60個

### タイプ別統計
- **Directory**: 49個（製品1 + Assy5 + SubAssy17 + SubSubAssy18 + Module8）
- **LeafProduct**: 60個
- **DirectoryDocument**: 15個
- **LeafProduct図面 (drawing_files)**: 180個（60部品 × 図面3つ）
- **LeafProductドキュメント (leaf_product_documents)**: 120個（60部品 × 2種類）

### 特徴
- 複雑な6階層構造
- 各DirectoryにDocument（図面・仕様書）が散在
- 全てのLeafProductに図面3つ + 仕様書 + 3Dモデルを含む
- モジュール単位での部品管理（Module階層）
- 左右対称の構造（フィンガーモジュール左/右など）
- 実際の産業用ロボットアームの構成に近い
